Release: v0.8.0 — Fancy Machine

This documentation set is updated for Coderive v0.8.0 (Fancy Machine). It includes a full language specification section with keyword, symbol, syntax, and runtime behavior details.

Getting Started

# Download curl -O https://github.com/DanexCodr/Coderive/releases/latest/download/coderive.jar # Create your first program echo 'unit hello share main() { out("Hello, Coderive!") }' > hello.cod # Run it java -jar coderive.jar hello.cod

Literate Specification (Comprehensive)

This section defines the current Coderive grammar surface and runtime behavior in a literate format: each area explains both what it is and how it is used.

Keywords

Visibility & Modules

share, local, unit, use

Controls declaration scope/visibility and source organization/imports.

Object & Type System

is, this, super, policy, with, type

Enables inheritance, instance references, and policy-virality contracts.

Control Flow

if, else, elif, for, of, to, by, break, skip, exit

Structured branching/looping with explicit range iteration syntax.

Core Types & Constants

int, float, text, bool, none, true, false

Built-in primitive/value literals and null-like sentinel.

Logic & Runtime Controls

all, any, get, set, control, unsafe, builtin

Quantifier-first logic and advanced low-level/runtime interaction controls.

Unsafe Primitive Types

i8, i16, i32, i64, u8, u16, u32, u64, f32, f64

Fixed-width numeric primitives enabled via unsafe declaration context.

Symbols & Operators

Assignment & Return

=, :=, ::, ::=, ~>

Includes typed/inferred declaration and slot-return assignment semantics.

Math & Comparison

+, -, *, /, %, ==, !=, >, <, >=, <=

Standard arithmetic/comparison operators with expression precedence.

Range/Sequence Operators

.., #, to, by

Defines range literals, formula shorthand, and step behavior.

Grouping & Access

( ), [ ], { }, ., , , :

Function calls, arrays/range slices, blocks, member access, and named slots.

Unary/Other Symbols

!, ?, |, &, $, _

Negation and reserved/special token support used by parser/runtime features.

Lambda & Self-Call

\\, <~(...), <~N(...), <~CONST(...)

Lambda syntax and recursive self-call syntax for methods and closures.

Core Syntax Behaviors

Variables, Constants, Types

name := "Coderive" age: int = 19 nums := [1 to 1M] MAX_RETRY: int = 5

ALL_CAPS identifiers act as constants and require initializers.

Functions, Slots, Lambdas

add := (\a, b) a + b local divide(a: int, b: int) :: quotient: float, remainder: int { ~> a / b, a % b }

Slot contracts are explicit; lambdas and methods support recursive self-call forms.

Loops & Ranges

for i of 1 to 10 by 2 { out(i) } for i of 1..32#*2 { out(i) } slice := arr[0 to 50 by 5]

Iteration uses for ... of; for ... in is legacy and unsupported.

Runtime & Semantic Behavior

Control Flow

Supports if/elif/else, loop control via break and skip, and early program/method exit via exit.

Quantifier-First Logic

if all[a > 0, b > 0] { ... } if any[isAdmin, isOwner] { ... }

Coderive uses all[]/any[] for explicit boolean composition.

Self Calls & Recursion

<~(...) performs self-call recursion; level form is supported for lambda parent access. The parser requires call parentheses; bare <~ is invalid.

Auto-Currying

Under-applied lambdas return closures. Extra arguments can chain into returned lambdas, enabling practical currying workflows.

Policies

Policy implementations are viral down inheritance chains; descendants must satisfy inherited policy methods.

Natural/Lazy Arrays

Large ranges are represented lazily with optimized iteration paths and formula-based loop behavior.
