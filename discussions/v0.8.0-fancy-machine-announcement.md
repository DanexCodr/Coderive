# Coderive v0.8.0 is live — Fancy Machine

We’re excited to announce **Coderive v0.8.0: Fancy Machine**.

## Highlights

- Full documentation/version refresh for v0.8.0
- New **comprehensive literate language specification** with both:
  - **What it is** (definition + intent)
  - **How it is used** (syntax + practical usage)
- Clarified semantics for advanced features:
  - self-call recursion (`<~(...)`, `<~N(...)`, `<~CONST(...)`)
  - lambda ancestry calls
  - auto-currying and closure chaining
  - return-slot semantics (`::`, `~>`)
  - policy inheritance behavior
- Documentation page upgrades:
  - integrated language specification section
  - client-side search
  - custom spec logo
- Updated changelog entry for v0.8.0 (Fancy Machine)
- Wiki-ready language specification content prepared

## Why v0.8.0 is powerful

Fancy Machine pushes Coderive forward in practical language design:

- **Readable correctness by default** — quantifier-first logic (`all[]`, `any[]`) removes ambiguous boolean precedence.
- **Structured multi-return APIs** — slot contracts and named extraction keep large codebases explicit and stable.
- **Scalable data workflows** — lazy/natural ranges allow huge sequence representations without materialization.
- **Expressive recursion tools** — direct self-calls and ancestor lambda calls make functional patterns concise and robust.
- **Safer constant conventions** — ALL_CAPS constants are explicit, initialized, and immutable by design.
